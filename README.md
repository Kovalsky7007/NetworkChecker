# NetworkChecker

NetworkChecker — инструмент диагностики сетевых блокировок и проблем доступа к сайтам.

В отличие от обычных утилит, показывает **где именно ломается соединение**:

DNS → TCP → TLS → CERT → HTTP

---

## 🚀 Возможности

* Полная проверка цепочки: DNS / TCP / TLS / HTTP / CERT
* Обнаружение DPI и RST-инъекций
* Определение MITM (подмена сертификата)
* Smart Arbitration (поддержка локальных CA, например НУЦ Минцифры)
* Проверка списков доменов из файлов
* Сетевой монитор соединений в реальном времени
* Двуязычный интерфейс (RU / EN)
* Генерация логов для техподдержки

---

## 🧠 В чём отличие

| Инструмент         | Ограничение                           |
| ------------------ | ------------------------------------- |
| ping               | Только ICMP                           |
| curl               | Только HTTP                           |
| online checker     | Проверка не из вашей сети             |
| **NetworkChecker** | Полная диагностика + причина проблемы |

---

## 📌 Пример результата

```
sberbank.ru   DEGRADED  DNS:OK TLS:OK HTTP:FAIL(RST) CERT:MITM
```

👉 Это значит:

* DNS работает
* TLS установлен
* но соединение сброшено → вероятен DPI

## ⚠️ Важно

* PING timeout ≠ сайт недоступен
* MITM может быть вызван антивирусом
* TLS:RST ≠ HTTP:RST

---

## 🔐 Smart Arbitration

Некоторые сайты используют нестандартные удостоверяющие центры (например банки).

Чтобы избежать ложного MITM:

1. Скачайте сертификаты
2. Поместите их в папку `Certs\`
3. Активируйте через меню (пункт 9)
