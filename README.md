# NetworkChecker

  <img src="global-network.png" alt="NetworkChecker" width="420">
</p>

**NetworkChecker** — инструмент диагностики сетевых блокировок и проблем доступа к сайтам для Windows.

В отличие от обычных утилит, он показывает **где именно ломается соединение**:

```
DNS → TCP → TLS → CERT → HTTP
```

---

## 🚀 Возможности

* Полная проверка цепочки: DNS / TCP / TLS / HTTP / CERT
* Обнаружение DPI и RST-инъекций
* Определение MITM (подмена сертификата)
* Smart Arbitration — поддержка локальных УЦ (например НУЦ Минцифры), чтобы банки и госсайты не помечались как MITM
* Проверка списков доменов из файлов `lists\`
* Сетевой монитор соединений в реальном времени
* Двуязычный интерфейс (RU / EN)
* Генерация логов для техподдержки

---

## 🧠 В чём отличие

| Инструмент         | Ограничение                           |
| ------------------ | ------------------------------------- |
| ping               | Только ICMP                           |
| curl               | Только HTTP                           |
| online checker     | Проверка не из вашей сети             |
| **NetworkChecker** | Полная диагностика + причина проблемы |

---

## 📌 Пример результата

```
sberbank.ru   DEGRADED  DNS:OK  TLS:OK  HTTP:FAIL(RST)  CERT:MITM
```

Это значит:

* DNS работает
* TLS установлен
* но соединение сброшено → вероятен DPI

## 🖥 Требования

* Windows 10 / 11
* PowerShell 5.1+
* Права администратора (скрипт сам запросит UAC при старте)
* Окно терминала **шириной ≥ 80 столбцов** (интерфейс рассчитан на 78 столбцов)

## ▶️ Запуск

**Проще всего:** двойной клик по **`NetworkChecker.bat`** — он сам найдёт скрипт и
запустит его, а скрипт запросит права администратора (UAC).

Альтернативы:

* ПКМ по `NetworkChecker_v1_14.ps1` → **Запустить с помощью PowerShell**
* или из консоли:

```powershell
powershell -ExecutionPolicy Bypass -File NetworkChecker_v1_14.ps1
```

> ⚠️ Имя скрипта — с подчёркиваниями: `NetworkChecker_v1_14.ps1`.
> Если PowerShell ругается на политику выполнения — используйте `.bat` или флаг
> `-ExecutionPolicy Bypass`, как выше.

## 🧭 Меню

| Пункт | Действие                                                            |
| ----- | ------------------------------------------------------------------ |
| `1`   | Сетевой монитор — живые TCP/UDP соединения (`Q`-выход `S`-сорт `F`-фильтр `P`-пауза) |
| `2`   | Проверка списков доменов из `lists\` (номера через запятую / `all`) |
| `6`   | Одиночная проверка одного или нескольких доменов                   |
| `7`   | Руководство пользователя                                           |
| `8`   | Переключить язык RU ⇄ EN                                            |
| `9`   | Управление сертификатами (Smart Arbitration)                       |
| `0`   | Выход                                                              |
| `L`   | Показать / скрыть легенду в правой колонке меню                     |
| `10`  | Debug-режим — отдельное окно с подробным логом                     |
| `12`  | Закрыть Debug-окно                                                 |

> Нумерация (1, 2, 6, 9…) с пропусками — историческая, так пункты не перемещались между версиями.

## ⚠️ Важно

* `PING timeout ≠ сайт недоступен` — многие банки и госсайты режут ICMP. Смотрите TLS и HTTP.
* `MITM` может быть вызван вашим антивирусом (Kaspersky/ESET перехватывают TLS).
* `TLS:RST` (DPI режет рукопожатие) ≠ `HTTP:FAIL(RST)` (DPI режет уже после TLS).

## 🔐 Smart Arbitration

Некоторые сайты (банки, госсервисы) используют сертификаты российских УЦ, которых нет в браузерных хранилищах — скрипт может ложно показать `MITM`.

Чтобы убрать ложную тревогу:

1. Скачайте корневые сертификаты: <https://www.gosuslugi.ru/crt>
2. Положите `.cer` файлы в папку `Certs\`
3. Активируйте их через меню (пункт `9`)
4. Запустите проверку снова — `MITM` сменится на `TRUSTED (File)`

---

## 📊 Логи

После любой проверки нажмите `S` — отчёт сохранится в `Logs\` с датой, вашим внешним IP, гео и всеми статусами. Файл можно отправить в техподдержку.
