<p align="center">
  <b>Диагностика сетевых блокировок для Windows — показывает, <i>где именно</i> ломается доступ к сайту.</b>
</p>

В отличие от обычных утилит (`ping`/`curl`), NetworkChecker проходит **всю цепочку**
соединения и называет **причину** проблемы — блокировка DPI, MITM, DNS-подмена или
просто сайт лёг:

```
DNS → TCP → TLS → CERT → HTTP
```

> 🟢 **Не разбираешься в сетях?** Открой простую инструкцию без терминов:
> [**docs/ПРОСТО.md**](docs/ПРОСТО.md) — как запустить и читать результаты за 2 минуты.

> ⬇️ **Скачать:** зелёная кнопка **Code → Download ZIP**, распакуй, двойной клик по
> `NetworkChecker.bat`. Прав администратора скрипт запросит сам.

## 🚀 Возможности

* Полная проверка цепочки: **DNS → TCP → TLS → CERT → HTTP**
* **Вывод простым языком** — каждый сайт получает понятный вердикт (`Доступен` /
  `Блок DPI/RST` / `Заглушка` / `MITM` / `Без страницы` / `Нет DNS`)
* Обнаружение **DPI** и RST-инъекций
* Точное определение **MITM** (по доверию цепочки + имени, без ложных тревог на CDN)
* **DNS-проверка** — сравнение системного/публичных/своих резолверов с **DoH-эталоном**,
  детект перехвата порта 53 и подмены, задержка по каждому резолверу
* **Smart Arbitration** — поддержка локальных УЦ (НУЦ Минцифры), чтобы банки и
  госсайты не помечались как MITM
* Проверка **списков доменов** из `lists\` — `all` даёт одну сводную таблицу
* **Планировщик** — ежедневный авто-скан через штатный `schtasks` (без служб)
* **Сетевой монитор** соединений в реальном времени (сортировка/фильтр/пауза)
* Двуязычный интерфейс (RU / EN), генерация логов для техподдержки

## 🗺 Roadmap — готовится релиз для Linux

🐧 **В работе — версия для Ubuntu и других дистрибутивов Linux.** Ядро диагностики
(TLS / сертификаты / MITM / HTTP / DoH) переносится на **PowerShell 7 (.NET)** и
работает кросс-платформенно.

Главный принцип переноса — **сохранить нативную связь с системными службами**, как
в Windows:

| Возможность | Windows (сейчас) | Linux (готовится) |
| ----------- | ---------------- | ----------------- |
| Авто-запуск по расписанию | Планировщик задач (`schtasks`) | **`cron` / systemd timer** |
| Права | UAC | `sudo` / root |
| Список соединений | `netstat` | `ss` |
| DNS-резолв | `Resolve-DnsName` | `dig` / системный резолвер |

Никаких сторонних демонов — только штатные средства ОС, как и на Windows.
Планируется единый код с ветками `$IsWindows` / `$IsLinux`, запуск через `pwsh`.

> Статус: в разработке. Следите за релизами и веткой проекта.

## 📌 Пример результата

Первая колонка — **вывод простым языком**, дальше техсырьё:

```
google.com     Доступен       DNS:OK  TLS:OK  HTTP:200       CERT:TRUSTED   IP:142.251.143.142
facebook.com   Заглушка       DNS:OK  TLS:OK  HTTP:FAIL(503) CERT:TRUSTED*  IP:188.186.146.208
whatsapp.com   Блок TLS       DNS:OK  TLS:FAIL HTTP:FAIL     CERT:FAIL      IP:157.240.205.60
gstatic.com    Без страницы   DNS:OK  TLS:OK  HTTP:FAIL(404) CERT:TRUSTED   IP:172.217.19.227

Итог: 4   Доступно:1  Без стр.:1  Блок:2  Нет DNS:0
```

Читается с первого взгляда: `Доступен` — работает, `Блок …`/`Заглушка` —
заблокирован, `Без страницы` — жив, просто нет веб-страницы (не блокировка).

## ▶️ Запуск

**Проще всего:** двойной клик по **`NetworkChecker.bat`** — он сам найдёт скрипт и
запустит его, а скрипт запросит права администратора (UAC).

Альтернативы:

* ПКМ по `NetworkChecker.ps1` → **Запустить с помощью PowerShell**
* или из консоли:

```powershell
powershell -ExecutionPolicy Bypass -File NetworkChecker.ps1
```

> Если PowerShell ругается на политику выполнения — используйте `.bat` или флаг
> `-ExecutionPolicy Bypass`, как выше.

**Авто-режим (без меню)** — для Планировщика задач или ручного запуска по расписанию:

```powershell
powershell -ExecutionPolicy Bypass -File NetworkChecker.ps1 -RunList all -Quiet
```

Проверит список(и) из `lists\` и сохранит отчёт в `Logs\` без интерактива.
Настроить ежедневный автозапуск проще через меню — пункт **3 (Планировщик задач)**.

## 🧭 Меню

| Пункт | Действие                                                            |
| ----- | ------------------------------------------------------------------ |
| `1`   | Сетевой монитор — живые TCP/UDP соединения (`Q`-выход `S`-сорт `F`-фильтр `P`-пауза) |
| `2`   | Проверка списков доменов из `lists\` — `all` или несколько номеров дают **одну сводную таблицу** (с дедупом), один номер — подробный разбор файла |
| `3`   | Планировщик задач — ежедневный авто-скан через `schtasks` (без служб) |
| `4`   | DNS-проверка — plaintext (системный/1.1.1.1/8.8.8.8/свои) против DoH-эталона, детект перехвата `:53` и подмены, задержка по резолверам |
| `6`   | Одиночная проверка одного или нескольких доменов                   |
| `7`   | Руководство пользователя                                           |
| `8`   | Переключить язык RU ⇄ EN                                            |
| `9`   | Управление сертификатами (Smart Arbitration)                       |
| `0`   | Выход                                                              |
| `L`   | Показать / скрыть легенду в правой колонке меню                     |
| `10`  | Debug-режим — отдельное окно с подробным логом                     |
| `12`  | Закрыть Debug-окно                                                 |

> Нумерация (1, 2, 6, 9…) с пропусками — историческая, так пункты не перемещались между версиями.

## 📂 Структура проекта

```
NetworkChecker.bat         — удобный запуск (двойной клик)
NetworkChecker.ps1         — сам скрипт
lists\                     — списки доменов (.txt, один домен на строку)
Logs\                      — отчёты (создаётся автоматически)
Certs\                     — .cer для Smart Arbitration
README.md                  — этот файл
docs\                      — вся подробная документация:
  ├ OVERVIEW.md            — текстовое описание проекта
  ├ MANUAL.md              — полное руководство пользователя
  ├ TECHNICAL.md           — техническое описание движка проверок
  ├ BUGS_AND_FIXES.md      — найденные проблемы, баги и их фиксы
  └ CHANGELOG.md           — история изменений
```

> Папки `Logs\`, локальный `Certs\active_certs.txt` и архив старых версий
> (`исходники\`) исключены из репозитория через `.gitignore`.

Формат файла списка (`lists\russia.txt`):

```
google.com
vk.com
# строки с # — комментарии, игнорируются
sberbank.ru
```

### Какие списки есть

| Список | Что внутри |
| ------ | ---------- |
| `google.txt` | сервисы и инфраструктура Google (вкл. YouTube) |
| `yandex.txt` | экосистема Яндекса |
| `telegram.txt` | Telegram: core, ДЦ, CDN, клиенты |
| `social.txt` | соцсети и мессенджеры (кроме Telegram) |
| `streaming.txt` | зарубежные видео/музыка (Netflix, Twitch, Spotify…) |
| `media_ru.txt` | российские медиа (Rutube, Kinopoisk, ivi, Okko…) |
| `services.txt` | зарубежные сервисы и e-commerce (Apple, Amazon, AliExpress…) |
| `CDN_global.txt` | глобальные CDN (Cloudflare, Akamai, Fastly…) |
| `global_status.txt` | сводный быстрый чек глобальной доступности |
| `russia_status.txt` | сводный быстрый чек доступности рунета |

> Каждый домен лежит ровно в одном тематическом списке. Два `*_status.txt` —
> это сводные «быстрые чеки» и **намеренно** повторяют популярные домены из
> тематических списков (помечено в шапке файла).

## ⚠️ Важно

* `PING timeout ≠ сайт недоступен` — многие банки и госсайты режут ICMP. Смотрите TLS и HTTP.
* `MITM` может быть вызван вашим антивирусом (Kaspersky/ESET перехватывают TLS).
* `TLS:RST` (DPI режет рукопожатие) ≠ `HTTP:FAIL(RST)` (DPI режет уже после TLS).

## 🔐 Smart Arbitration

Некоторые сайты (банки, госсервисы) используют сертификаты российских УЦ, которых нет в браузерных хранилищах — скрипт может ложно показать `MITM`.

Чтобы убрать ложную тревогу:

1. Скачайте корневые сертификаты: <https://www.gosuslugi.ru/crt>
2. Положите `.cer` файлы в папку `Certs\`
3. Активируйте их через меню (пункт `9`)
4. Запустите проверку снова — `MITM` сменится на `TRUSTED (File)`

## 📊 Логи
