## v1.14.2 — «плывущий» интерфейс и чистка кода

Жалоба пользователя: «там ошибки есть и интерфейс плывёт».

<a name="bug-01"></a>
### BUG-01 — Меню переносится на узкой консоли (главная причина «плыва»)

- **Симптом.** Главное меню «разъезжается»: правая колонка с легендой съезжает,
  строки ломаются.
- **Причина.** Двухколоночный рендер был шириной `LeftW(38) + легенда(42–43) =
  80–81` столбец. На стандартной консоли шириной **80** запись 80+ символов
  вызывает автоперенос строки → каждая строка легенды переносится, всё съезжает.
- **Фикс.** Сузили до `LeftW=36 + легенда 42 = 78` столбцов (< 80, переноса нет).
  Левый столбец и сепараторы подогнаны под 36; заголовок укорочен, чтобы метка
  `[DEBUG]` влезала.
- **Проверено.** Рендер-тест `Show-IPInfo`: максимальная ширина строки = 78,
  переноса нет ни в RU, ни в EN.

<a name="bug-02"></a>
### BUG-02 — Рамки легенды расходятся на 1–2 символа

- **Симптом.** Правая граница `│` в блоке легенды «гуляет» по вертикали.
- **Причина.** Рамки рисовались **вручную** строками-литералами. Замер показал:
  в RU строки 10 и 12 были длиной 41 вместо 42; в EN часть строк 43, часть 42.
  Любое расхождение по ширине ломает вертикаль границы.
- **Фикс.** Введена функция **`Format-Box`**: принимает заголовок и массив строк,
  программно дополняет каждую до фиксированной `Inner`-ширины и добавляет рамку
  `┌│└─`. Выравнивание теперь гарантировано арифметикой, а не подсчётом пробелов.
  `$global:LegendData` пересобран через `Format-Box`.
- **Проверено.** Прогон `Format-Box`/`LegendData` из реального файла: **все**
  строки рамки ровно 42 символа (и RU, и EN), ничего не обрезано.

<a name="bug-03"></a>
### BUG-03 — Дубликат функции Show-Legend

- **Симптом.** Поведение легенды по кнопке `L` не соответствовало «лучшему»
  определению.
- **Причина.** Функция `Show-Legend` была **объявлена дважды** (строки ~1311 и
  ~1335). В PowerShell побеждает последнее определение — первое (с валидацией
  цвета и `ReadKey`) затиралось вторым (с `Read-Host`).
- **Фикс.** Дубликат удалён, оставлено одно определение. (Позже, в v1.14.3,
  overlay вообще убран — роль легенды выполняет правый столбец меню.)
- **Проверено.** `grep`: `function Show-Legend` встречается 1 раз; парсер чист.

<a name="bug-04"></a>
### BUG-04 — Мёртвый код Show-SideHelp (~170 строк)

- **Симптом.** В файле большой блок с кривыми ASCII-рамками.
- **Причина.** Функция `Show-SideHelp` была определена, но **нигде не вызывалась**
  (убрана из режимов ещё в v1.14.1, тело осталось). Лишний код + рассинхрон.
- **Фикс.** Функция полностью удалена.
- **Проверено.** `grep`: 0 определений и 0 вызовов (остались только упоминания
  в комментариях-истории).

<a name="bug-05"></a>
### BUG-05 — Браузеры lists\ и Certs\ без правой границы

- **Симптом.** В списках файлов рамка «открыта» справа: верх/низ имеют `┐`/`┘`,
  а строки внутри — нет правой `│`. Для Certs верх и низ ещё и разной ширины
  (64 против 63).
- **Причина.** Строки печатались форматом без закрывающей границы и без
  выравнивания по ширине рамки.
- **Фикс.** Каждая строка дополняется до внутренней ширины (59) и закрывается
  `│`. Рамки Certs пересобраны программно (одинаковый `Inner=59`).
- **Проверено.** Замер: top/row/bottom = 63 для обоих браузеров.

<a name="bug-06"></a>
### BUG-06 — Неверное имя файла в Руководстве

- **Симптом.** Команда запуска из встроенного Руководства не нашла бы скрипт.
- **Причина.** В тексте значилось `NetworkChecker_v1.7.ps1` и
  `NetworkChecker_v1.14.ps1` (с точкой), а файл на диске —
  `NetworkChecker_v1_14.ps1` (с подчёркиваниями).
- **Фикс.** Все ссылки приведены к реальному имени `NetworkChecker_v1_14.ps1`.

<a name="bug-07"></a>
### BUG-07 — Битые ссылки на документацию, отсутствует MANUAL.md

- **Симптом.** README ссылался на `docs/TECHNICAL.md` и `docs/MANUAL.md`, но
  таких путей не было, а `MANUAL.md` вообще отсутствовал.
- **Фикс.** Переписан `README.md`, создан полноценный `MANUAL.md`. (В этой ревизии
  вся документация перенесена в каталог `docs/`, ссылки обновлены.)

## v1.14.3 — сетевой монитор, легенда, списки

Жалоба пользователя: «сетевой монитор очень длинный, добавь сортировку и
исключение; автообновление сбивает курсор — это бесит; легенда должна
включаться/выключаться и быть привязана к правому углу».

<a name="bug-08"></a>
### BUG-08 — Монитор мерцает и сбивает курсор

- **Симптом.** При автообновлении (каждые 3 с) экран дёргается, курсор прыгает,
  мигание.
- **Причина.** Каждый тик выполнялся `Clear-Host` с полной перерисовкой —
  отсюда вспышка и сброс позиции.
- **Фикс.** Один `Clear-Host` при входе; далее каждый кадр рисуется через
  `[Console]::SetCursorPosition(0,0)` поверх старого, строки дополняются до
  ширины окна, «хвост» прошлого кадра затирается пробелами. Курсор скрывается
  (`CursorVisible=$false`) и восстанавливается в `finally`.
- **Проверено.** Парсер чист; логика рендера кадра проверена на мок-данных.

<a name="bug-09"></a>
### BUG-09 — Монитор слишком длинный, нет фильтра и сортировки

- **Симптом.** Список соединений огромный, уезжает за экран, неудобно.
- **Фикс.** Добавлены интерактивные клавиши:
  - `S` — сортировка: Состояние → Процесс → Гео → Адрес;
  - `F` — фильтр/исключение: Все → Активные → Без LISTEN → Только внешние
    (последний отбрасывает loopback, `10.*`, `192.168.*`, `172.16–31.*`,
    `*::1*`, `fe80*`);
  - `P` — пауза автообновления.
  Тело таблицы обрезается по высоте окна (`WindowHeight`), лишнее показано как
  «… ещё N».
- **Проверено.** Юнит-тест фильтров (4 режима) и сортировок (Состояние/Гео) на
  мок-наборе соединений — результаты корректны.

<a name="bug-10"></a>
### BUG-10 — Легенду нельзя выключить, обрезается Geo

- **Симптом.** Легенда всегда занимает правый столбец; длинная строка Geo
  обрезается (`Netherlands, Amsterda…`).
- **Фикс.** Введён `$global:ShowLegend`; кнопка `L` в меню переключает легенду
  вкл/выкл (вместо прежнего overlay). В режиме **выкл** меню одноколоночное во
  всю ширину — IP и Geo видны полностью. Состояние видно в подсказке
  (`[L-легенда:вкл/выкл]`).
- **Проверено.** Рендер-тест обоих режимов: ON — 78 столбцов, выровнено;
  OFF — Geo отображается целиком.

<a name="bug-11"></a>
### BUG-11 — Битый ввод доменов уходит в DNS как мусор

- **Симптом.** Строка вида `://google.com` из списка превращалась в `:` и
  отправлялась в DNS-резолв (бесполезный запрос, ложный фейл).
- **Причина.** `Sanitize-Domain` снимал только `https://` и `//`, но не `://`
  и не валидировал результат.
- **Фикс.** Санитайзер ужесточён: снятие любой схемы `*://`, ведущих `:`/`/`,
  порта `:NNNN`, и финальная валидация — домен должен содержать точку и только
  `[a-zA-Z0-9.-]`. Иначе возвращается `""` и строка отбрасывается фильтром.
- **Проверено.** Тест-кейсы: `://google.com → google.com`, `vk.com:443/path →
  vk.com`, `:`/`-`/`localhost → ""` (отбраковка), punycode сохраняется.

<a name="bug-12"></a>
### BUG-12 — Мусор в списках доменов

- **Симптом.** В `lists/google.txt` — 4 строки `://google.com` и дубль
  `googlevideo.com`; в `lists/services.txt` — строка `://apple.com`.
- **Фикс.** Списки вычищены вручную (помимо защиты в санитайзере из BUG-11).
- **Проверено.** Просмотр файлов; остальные списки проверены — чистые.

<a name="bug-13"></a>
### BUG-13 — Список Telegram с кириллическим именем

- **Симптом.** Файл `lists/телеграмм.txt` (кириллица) выбивался из соглашения
  об именах (остальные — латиницей) и не был под контролем версий.
- **Фикс.** Переименован в `lists/telegram.txt`, добавлен завершающий перевод
  строки, включён в репозиторий.

## v1.14.4 — удобство, локализация, авто-режим

Задача пользователя: «приведи в порядок, сделай понятнее, сохрани функционал,
исправь баги; всё системными средствами, без Python».

### BUG-14 — `-DebugMode` при запуске не включал Debug

- **Симптом.** Запуск `... -DebugMode` поднимал Verbose-вывод, но в меню не было
  метки `[DEBUG]`, а пункт 10 переключался «от пустого значения».
- **Причина.** Параметр `-DebugMode` выставлял только `$VerbosePreference`;
  `$global:DebugMode` (который читают меню и пункт 10) оставался `$null`.
- **Фикс.** `$global:DebugMode = [bool]$DebugMode` сразу после `param()`.

### BUG-15 — параметры терялись при UAC-элевации

- **Симптом.** При перезапуске с правами администратора `-DebugMode` (а теперь и
  `-RunList`/`-Quiet`) не доходили до повышенного процесса.
- **Фикс.** Аргументы пробрасываются в `Start-Process … RunAs`.

### BUG-16 — захардкоженный русский в EN-режиме

- **Симптом.** В English-режиме запрос количества пингов, `DOWN (DNS failed)`,
  итоговый `Status`, предупреждение MITM, слово «дней», подсказки выбора и
  «Лог сохранён» всё равно выводились по-русски.
- **Фикс.** Добавлены ключи в таблицу локализации `$global:T` (RU/EN), строки
  заменены вызовами `T '<key>'`. Парность ключей RU=EN проверена.

### Чистка — `Get-NetSpeed` молотил впустую

- Функция каждый тик монитора (3 с) перебирала **все** процессы и складывала
  `WorkingSet64` в глобальный хеш, который нигде не читался. Заменена честной
  заглушкой `return @{}` — поведение (KB/s = «-») не изменилось, нагрузка ушла.

### Версия в одну константу

- Раньше строка `v1.14` была захардкожена в ~15 местах вывода. Введён
  `$script:Version` / `$script:VersionTag` — единственная точка правки при релизе.
  Рамки Руководства (`╔══╗`) пересобраны программно (`Inner=66`), чтобы
  выравнивание не зависело от длины версии.

### Переименование файла

- `NetworkChecker_v1_14.ps1` → `NetworkChecker.ps1`. Версия больше не в имени
  файла (она в `$script:Version` и CHANGELOG). `.bat` ищет новый файл, с откатом
  на легаси-маску `NetworkChecker_v*.ps1`.

### Авто-режим и Планировщик задач (новое)

- Параметры `-RunList <имя|all> -Quiet` гоняют проверку списков без меню и пишут
  отчёт в `Logs\`. На этом построен пункт меню **3** — создаёт ежедневную задачу
  через штатный `schtasks` (`/SC DAILY /RL HIGHEST`), без служб и Python.
  Управление: создать/пересоздать/удалить, показ статуса задачи.

<a name="v1145"></a>
## v1.14.5 — точность вердикта по сертификату

Задача пользователя: «диагностика должна быть максимально точной».

### BUG-17 — массовые ложные MITM на легитимных CDN

- **Симптом.** Десятки доменов Яндекса и Google (`yastatic.net`, `yastat.net`,
  `ya.ru`, `an.yandex.ru`, `googlevideo.com`, `youtube.com`…) помечались как
  `MITM`, хотя перехвата не было.
- **Причина.** Вердикт `MITM` ставился по **одному** несовпадению имени
  сертификата с доменом и **до** проверки доверия цепочки (`return MITM`
  выполнялся раньше, чем код доходил до `chain.Build`). Плюс разбор SAN был
  неточным. В итоге CDN-серт с валидной публичной цепочкой, но другим именем
  в CN (напр. `yastatic.net` обслуживается сертом `*.yastatic-net.ru`) ложно
  метился перехватом.
- **Фикс.** Вердикт перестроен на два независимых факта:
  - `hostMatch` — корректная проверка имени: CN + все DNS-имена из SAN (читаются
    по OID `2.5.29.17`, без зависимости от локали Windows) с правилами wildcard
    (`*.x` покрывает ровно один лейбл);
  - `chainOK` — построение цепочки до доверенного корня Windows.

  Матрица решений:
  | имя | цепочка | вердикт |
  | --- | ------- | ------- |
  | совпало | доверена | `TRUSTED` |
  | совпало | не доверена | арбитраж `Certs\` → `TRUSTED (File)`, иначе `SELF-SIGN`/`!CHAIN` |
  | не совпало | доверена | `TRUSTED (CDN)` — публичный серт подделать нельзя, это не перехват |
  | не совпало | не доверена | **`MITM`** — недоверенный серт на чужое имя (реальный перехват) |

- **Проверено.** Live-тест на реальных сертификатах: `yastatic.net`
  (CN=`*.yastatic-net.ru`), `ya.ru` (CN=`*.yandex.tr`), `google.com`,
  `github.com`, `cloudflare.com`, `apple.com` — все `hostMatch=True`,
  `chainOK=True` → `TRUSTED`. Старый код давал на части из них ложный MITM.

<a name="known"></a>
## Известные ограничения (не баги)

- **Колонка KB/s в мониторе** показывает «-». Реальный per-process трафик через
  perf-счётчики/WMI слишком медленный для 3-секундного тика, поэтому оставлена
  заглушка. *Идея на будущее:* считать дельту по `Get-NetTCPConnection` +
  ETW/`Get-Counter` в фоне, либо показывать суммарную скорость адаптера.
- **Только Windows / PowerShell 5.1** — используются `netstat`, Windows API,
  .NET 4.x.
- **Geo и внешний IP** запрашиваются у публичных API (`ip-api.com`, `ipify` и
  др.) — при их недоступности показывается «Geo unavailable».
- **ICMP** часто режется провайдером — `PING:timeout` не равен недоступности.

<a name="how-tested"></a>
## Как это проверялось

Подход к правкам большого (2200+ строк) файла без CI:

1. **Бэкап** оригинала перед началом (`NetworkChecker_v1_14.ps1.bak`).
2. **Парсер** после каждой структурной правки:
   `[System.Management.Automation.Language.Parser]::ParseFile(...)` — ловит
   синтаксис до запуска.
3. **Рендер-тесты** изменённых функций: `Format-Box`, `LegendData`, `Show-IPInfo`
   (оба режима легенды) извлекались из реального файла и прогонялись, ширина
   строк измерялась программно.
