# NetworkChecker — Руководство пользователя

Версия v1.14 · Windows 10/11 · PowerShell 5.1+

Это полная версия руководства. Краткое — в самом скрипте (пункт меню `7`).

---

## 1. Что это и зачем

NetworkChecker — инструмент сетевой диагностики. Он проверяет доступность сайтов
**из вашей сети** и показывает, на каком этапе ломается соединение:

* **DNS** — резолвится ли домен в IP-адрес
* **TCP/TLS** — открыт ли порт 443 и проходит ли защищённое рукопожатие
* **HTTP** — отвечает ли веб-сервер и каким кодом
* **CERT** — валиден ли сертификат, нет ли подмены (MITM)
* **PING** — задержка и потери пакетов (ICMP)

Полезен для: диагностики блокировок (DPI/RST/SNI), проверки доступности сервисов,
подготовки отчётов для технической поддержки.

---

## 2. Требования

* Windows 10 или 11
* PowerShell 5.1 и новее
* Права администратора — нужны для сетевого монитора (`netstat`, счётчики).
  Скрипт сам поднимет UAC при запуске.
* Терминал шириной **не менее 80 столбцов** — интерфейс рассчитан на 78.

## 3. Установка и структура папок

Распакуйте проект в любую папку. Рядом со скриптом должны лежать (часть создаётся сама):

```
NetworkChecker_v1_14.ps1   ← сам скрипт
lists\                     ← списки доменов (.txt)
│   ├── russia_status.txt      один домен на строку, без https://
│   ├── global_status.txt      строки с # — комментарии, игнорируются
│   └── (любые .txt)           скрипт найдёт их автоматически
Logs\                      ← создаётся автоматически, сюда пишутся отчёты
Certs\                     ← опционально, .cer для Smart Arbitration
│   ├── russian_trusted_root_ca.cer
│   └── active_certs.txt       список активных (создаётся скриптом)
```

**Формат файла списка** (`lists\russia.txt`):

```
google.com
vk.com
# это комментарий — строка игнорируется
sberbank.ru
```

## 4. Запуск

* ПКМ по файлу → **Запустить с помощью PowerShell** (UAC поднимется сам), или
* в консоли:

```powershell
powershell -ExecutionPolicy Bypass -File NetworkChecker_v1_14.ps1
```

При первом запуске вверху появятся ваш локальный IP, внешний IP и геолокация —
они запрашиваются один раз и кэшируются.

## 5. Пункты меню

| Пункт | Действие |
| ----- | -------- |
| `1`   | **Сетевой монитор** — живые TCP/UDP соединения, обновление каждые 3 с. `Q`-выход, `S`-сортировка, `F`-фильтр, `P`-пауза. |
| `2`   | **Проверка списков** — показывает все `.txt` из `lists\`; введите номера через запятую или `all`. |
| `6`   | **Одиночная проверка** — подробная диагностика одного/нескольких доменов. |
| `7`   | **Руководство** (краткое). |
| `8`   | **Переключить язык** RU ⇄ EN (мгновенно, без перезапуска). |
| `9`   | **Сертификаты** — управление Smart Arbitration. |
| `0`   | **Выход**. |
| `L`   | **Легенда** — показать/скрыть расшифровку статусов в правой колонке меню. Выкл — чистое одноколоночное меню (IP/Geo видны полностью). Работает в любой раскладке (`L`/`Л`). |
| `10`  | **Debug-режим** — открывает отдельное окно с подробным Verbose-логом в реальном времени. |
| `12`  | **Закрыть Debug-окно**. |

> Номера идут с пропусками (1, 2, 6, 9…) — это историческая нумерация,
> чтобы пункты не «переезжали» между версиями.

### 5.1. Сетевой монитор (1)

Показывает все активные соединения компьютера. Колонки:
`процесс / состояние / удалённый адрес / страна / скорость KB/s / PID`.

Обновляется каждые 3 секунды **без мерцания** (экран перерисовывается на месте,
курсор не прыгает). Если список длинный — он обрезается по высоте окна, лишнее
показано счётчиком «… ещё N».

**Клавиши управления (нажимать прямо в мониторе):**

* `Q` — выход в меню
* `S` — сменить сортировку: Состояние → Процесс → Гео → Адрес
* `F` — сменить фильтр (исключение): Все → Активные → Без LISTEN → Только внешние
* `P` — пауза/продолжить автообновление (удобно, чтобы спокойно прочитать список)

Фильтр «Только внешние» убирает локальный шум (loopback, `192.168.*`, `10.*`,
`172.16–31.*`, прослушиваемые порты) и оставляет только реальные внешние соединения.

Цвета состояний:

* `ESTAB` (зелёный) — активное соединение
* `SYN-S` (красный) — ждёт ответа (возможен блок)
* `LISTN` (голубой) — порт слушает входящие
* `UDP53` (жёлтый) — DNS-запрос

### 5.2. Проверка списков (2)

Выберите файлы по номерам (`1,3`) или `all`. Идёт прогресс-бар, по завершении —
итоговая строка `UP / DEGRADED / DOWN` и предложение сохранить лог (`S`).

### 5.3. Одиночная проверка (6)

Ввод: `google.com` или `google.com,vk.com,sberbank.ru` (через запятую).
Показывает DNS, Ping (avg/median/min/max/loss), TLS, сертификат и HTTP по каждому домену.

## 6. Как читать результаты

Строка результата в режиме списка выглядит так:

```
domain.ru   STATUS   DNS:OK  TLS:OK  HTTP:200  PING:25ms  CERT:TRUSTED  IP:...
```

### DNS — получен ли IP домена
* `DNS:OK` — домен резолвится
* `DNS:FAIL` — возможно, блок на уровне DNS-сервера

### TLS — открыт ли защищённый порт 443
* `TLS:OK` — TCP установлен, TLS-рукопожатие прошло
* `TLS:FAIL` — порт закрыт или провайдер вмешивается в соединение

### HTTP — ответ веб-сервера
* `HTTP:200` — сайт отвечает нормально
* `HTTP:301/302` — редирект (обычно нормально)
* `HTTP:FAIL(403)` — сервер доступен, но доступ закрыт (часто блок по IP)
* `HTTP:FAIL(timeout)` — соединение есть, ответа нет
* `HTTP:FAIL(RST)` — соединение сброшено (возможен DPI)

### PING — задержка (ICMP)
Показывает `avg` (среднее) и `median` (медиана). Медиана надёжнее — она не
учитывает единичные выбросы. Если `avg` сильно выше `median` — был один долгий
пакет, а не общий лаг.

> **Важно:** многие серверы блокируют ICMP. `PING:timeout` **не** означает,
> что сайт недоступен — смотрите TLS и HTTP.

### CERT — статус сертификата
* `TRUSTED` — цепочка доверия проверена
* `TRUSTED (CDN)` — сертификат от CDN (другой домен, тот же владелец)
* `TRUSTED (File)` — подтверждён файлом из `Certs\` (Smart Arbitration)
* `EXPIRED` — сертификат просрочен
* `SELF-SIGN` — самоподписанный, не из публичного УЦ
* `!CHAIN` — цепочка доверия прервана
* `MITM` — CN сертификата не совпадает с доменом (возможен перехват)
* `DPI/RST` — соединение сброшено при попытке установить TLS

### Итоговый статус строки
* `UP` (зелёный) — DNS + TLS + HTTP + CERT все в порядке
* `DEGRADED` (жёлтый) — сервер виден, но что-то мешает (смотрите колонки)
* `DOWN` (красный) — DNS не прошёл или TCP полностью недоступен

> `TLS:RST` (DPI режет само рукопожатие) — это **не** то же самое, что
> `HTTP:FAIL(RST)` (DPI режет уже после установленного TLS).

## 7. Smart Arbitration — «обеление» банков

**Проблема.** Российские банки и госсервисы используют сертификаты НУЦ Минцифры,
которых нет в стандартных браузерных хранилищах. Без них скрипт показывает `MITM` —
это ложная тревога.

**Решение:**

1. Скачайте корневые сертификаты НУЦ: <https://www.gosuslugi.ru/crt>
   (нужны `russian_trusted_root_ca.cer` и `russian_trusted_root_ca_gost_2025.cer`).
2. Положите `.cer` в папку `Certs\` рядом со скриптом.
3. В меню нажмите `9`, выберите нужные сертификаты:
   * номера через запятую — включить/выключить (toggle)
   * `all` — активировать все
   * `0` — сбросить все
4. Запустите проверку снова — `MITM` сменится на `TRUSTED (File)`.

**Как экспортировать сертификат из браузера:** откройте сайт → клик по замку →
«Сертификат» → «Подробности» → выберите корневой УЦ в иерархии → «Экспорт» → DER (.cer).

## 8. Логи и отчёты

После любой проверки нажмите `S` — отчёт сохранится в `Logs\` под именем
`Log_<Тип>_<дата>_<время>.txt`. Файл содержит дату, ваш внешний IP, геолокацию и
все статусы. Его можно целиком отправить в техподдержку.

Для одиночной проверки лог подробнее — вертикальный формат:

```
--- google.com [10:30:15] ---
DNS    : OK  IP: 142.250.74.46
Ping   : avg=18ms  median=14ms  min=12ms  max=45ms  loss=0%
CERT   : 2025-06-01 (40 дней)  Trust:TRUSTED
Статус : UP
```

## 9. Частые вопросы / неполадки

* **Все домены показывают `CERT:FAIL`.** Проверьте системные часы — сбитое время
  ломает проверку сертификатов. Скрипт предупреждает о дрейфе часов при старте.
* **Банк показывает `MITM`.** Это ожидаемо без Smart Arbitration — см. раздел 7.
* **`MITM` на обычном сайте.** Возможно, ваш антивирус (Kaspersky/ESET) перехватывает
  TLS. Это не блокировка провайдера.
* **`PING:timeout`, но сайт работает.** Нормально: ICMP часто режется. Ориентируйтесь
  на TLS и HTTP.
* **Интерфейс выглядит «рвано».** Расширьте окно терминала минимум до 80 столбцов.
* **Нужны подробности по ошибке.** Включите Debug (пункт `10`) — откроется окно с
  Verbose-логом; лог также пишется в `Logs\debug_verbose.log`.
