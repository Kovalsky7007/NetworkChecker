# Changelog

Все заметные изменения проекта. Формат основан на здравом смысле, версии — по
истории разработки скрипта.

## v1.14.5

### Точность диагностики сертификатов (главное)
- **Переписана логика MITM.** Раньше вердикт `MITM` ставился по одному
  несовпадению имени сертификата с доменом — и **до** проверки доверия цепочки.
  Из-за этого CDN-серты Яндекса/Google (валидная публичная цепочка, но другое
  имя в CN, напр. `yastatic.net` через `*.yastatic-net.ru`) массово помечались
  ложным MITM.
- Теперь вердикт опирается на **два независимых факта**: совпадение имени
  (`hostMatch`, корректный разбор CN + SAN с правилами wildcard) и доверие
  цепочки (`chainOK`, построение до доверенного корня Windows). `MITM` ставится
  **только** когда имя не совпало **и** цепочка не доверена — т.е. подсунут
  недоверенный серт на чужое имя (стаб блокировки, антивирус со своим CA).
  Публично доверенный серт на другое имя подделать нельзя → это CDN, не перехват.
- Разбор SAN теперь читается по OID `2.5.29.17` и не зависит от локали Windows
  (метка `DNS Name`/`DNS-имя` игнорируется, берётся само имя).
- Итог: меньше ложных MITM на легитимных CDN, реальные перехваты (напр. кластер
  заблокированных доменов на одном стаб-IP) по-прежнему детектируются.
- Новый ярлык **`TRUSTED*`**: валидный публичный серт, но имя не совпало с
  доменом. Это не перехват (такой серт не подделать), но может быть как
  настоящий CDN, так и DNS-редирект на стаб-заглушку (напр. блок Meta, когда
  несколько доменов резолвятся на один IP с `HTTP:FAIL(503)`). Настоящий CDN
  «того же владельца» (база CN = база домена) остаётся `TRUSTED (CDN)`.

## v1.14.4

### Исправления
- `-DebugMode` при запуске теперь действительно включает Debug: глобальный флаг
  `$global:DebugMode` синхронизируется с параметром (раньше метка `[DEBUG]` в меню
  не появлялась, а пункт 10 переключался «от пустого значения»).
- Флаг `-DebugMode` пробрасывается при запросе прав администратора (UAC) — раньше
  терялся при перезапуске.
- Локализация: запрос количества пингов, строка `DOWN (DNS failed)`, итоговый
  `Status` и подсказки выбора списков больше не выводятся всегда по-русски —
  в EN-режиме теперь корректный английский.

### Чистка
- `Get-NetSpeed` больше не перебирает все процессы каждый тик впустую (результат
  всё равно не использовался) — оставлена честная заглушка. Колонка KB/s
  по-прежнему показывает «-» (см. «Известные ограничения»).

### Сводная проверка списков
- Пункт 2: при выборе `all` (или нескольких номеров) теперь выводится **одна
  общая таблица** по всем выбранным спискам с единым итогом, а не отдельная
  таблица на каждый файл. Домены дедуплицируются между списками — повторы
  (напр. из сводных `*_status`) проверяются один раз.

### Косметика вывода
- Колонка `IP:` в таблице проверки больше не «уезжает»: поле HTTP расширено
  до 13 символов (вмещает `FAIL(timeout)`/`FAIL(TLS-err)`).
- `PING:timeout` и `PING:N/A` выводятся без лишнего «ms» (хелпер `Format-Ms`).
  То же в подробном выводе одиночной проверки (`avg=timeout` вместо `avg=timeoutms`).

### Списки доменов (`lists\`)
- Наведён порядок: каждый домен теперь живёт в **одном** тематическом списке —
  убраны межсписочные дубли (YouTube-домены только в `google`, Netflix/Spotify/
  Twitch — в `streaming`, Apple — в `services`, Telegram — в `telegram`,
  Kinopoisk — в `media_ru` и т.д.).
- `global_status.txt` и `russia_status.txt` оставлены как **сводные быстрые
  чеки** (по одному представителю на сервис) — в шапке файла помечено, что
  повтор популярных доменов здесь намеренный.
- `tdesktop.com` перенесён в `telegram.txt`, `akamaihd.net` — в `CDN_global.txt`.

### Структура проекта
- Удалены из репозитория лишние артефакты (бэкап `.bak`, архив `исходники/`,
  рабочие `чаты_из_архива/`) — перенесены в `_на_удаление/`.

## v1.14.3

### Сетевой монитор
- Анти-мерцание: экран перерисовывается на месте (`SetCursorPosition`) вместо
  `Clear-Host` на каждом тике — курсор больше не прыгает при автообновлении.
- Сортировка (`S`): Состояние → Процесс → Гео → Адрес.
- Фильтр/исключение (`F`): Все → Активные → Без LISTEN → Только внешние.
- Пауза (`P`) — заморозить автообновление.
- Длинный список обрезается по высоте окна (показывается «… ещё N»).

### Меню
- Кнопка `L` теперь переключает легенду (вкл/выкл) в правой колонке, а не
  открывает отдельный overlay. В режиме «выкл» меню одноколоночное и IP/Geo
  видны полностью.

### Прочее
- `Sanitize-Domain` ужесточён: битый ввод вида `://google.com`, `:`, `-` больше
  не уходит в DNS-резолв; добавлена валидация домена.
- Очищены списки `lists/google.txt` и `lists/services.txt` (мусор и дубликаты).
- Список Telegram переименован `телеграмм.txt` → `lists/telegram.txt`.
- Удобный запуск: `NetworkChecker.bat` (двойной клик).
- Документация собрана в каталог `docs/` (`OVERVIEW.md`, `MANUAL.md`,
  `TECHNICAL.md`, `BUGS_AND_FIXES.md`, `CHANGELOG.md`); `README.md` остался в корне.
