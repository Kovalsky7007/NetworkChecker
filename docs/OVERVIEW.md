## 1. Что это

**NetworkChecker** — консольный инструмент диагностики сетевых блокировок и
проблем доступа к сайтам под Windows. Написан на PowerShell 5.1 одним файлом,
без внешних зависимостей.

Ключевая идея: обычные утилиты отвечают «да/нет» на вопрос «работает ли сайт».
NetworkChecker отвечает на вопрос **«где именно ломается соединение»**, проходя
всю цепочку по шагам:

```
DNS → TCP → TLS → CERT → HTTP
```

и показывая, на каком звене всё рвётся (и почему: DPI, RST-инъекция, MITM,
блок по IP, просроченный сертификат и т.д.).

---

## 2. Зачем (мотивация)

| Инструмент      | Что умеет                | Чего не хватает                       |
| --------------- | ------------------------ | ------------------------------------- |
| `ping`          | ICMP-эхо                 | ничего про TLS/HTTP; ICMP часто режут |
| `curl`          | HTTP-запрос              | не различает DNS/TLS/DPI причины      |
| online-checkers | доступность «снаружи»    | проверяют не из вашей сети            |
| **NetworkChecker** | вся цепочка + причина | —                                     |

Типичные сценарии:

- понять, заблокирован ли сайт у провайдера и каким способом (DNS/SNI/DPI);
- отличить реальную недоступность от ложной (ICMP-таймаут, антивирусный MITM);
- собрать отчёт для техподдержки одним файлом;
- наблюдать живые соединения компьютера (сетевой монитор).

---

## 3. Как это работает (цепочка проверок)

Для каждого домена:

1. **DNS** — `Resolve-DnsName`, берётся первая A-запись. Нет IP → `DOWN`.
2. **TCP + TLS** — ручное рукопожатие через `TcpClient` + `SslStream`
   (`Tls12|Tls13`). Отдельно ловится `ConnectionReset` → метка `DPI/RST`.
3. **CERT** — разбор сертификата:
   - CN/SAN сверяются с доменом → детекция `MITM`;
   - CDN-сертификаты (другой TLD, тот же владелец) → `TRUSTED (CDN)`;
   - локальные УЦ из `Certs\` (Smart Arbitration) → `TRUSTED (File)`;
   - цепочка доверия через `X509Chain` → `TRUSTED` / `!CHAIN` / `EXPIRED` /
     `SELF-SIGN`.
4. **HTTP** — `Invoke-WebRequest` с полным набором заголовков Chrome (обход WAF),
   при неудаче — fallback на `http://`. Причина фейла нормализуется
   (`FAIL(403)`, `FAIL(RST)`, `FAIL(timeout)`).
5. **PING** — `System.Net.NetworkInformation.Ping`, считается avg/median/min/max
   и процент потерь. Медиана — основной показатель (устойчива к выбросам).

Итоговый статус: `UP` (всё хорошо) / `DEGRADED` (сервер виден, но что-то мешает)
/ `DOWN` (DNS не прошёл или TCP закрыт).

## 4. Возможности

- Полная диагностика цепочки DNS/TCP/TLS/HTTP/CERT с указанием причины.
- Детекция DPI/RST-инъекций и MITM (подмена сертификата).
- **Smart Arbitration** — поддержка локальных УЦ (НУЦ Минцифры и др.), чтобы
  банки и госсайты не помечались ложным `MITM`.
- Проверка списков доменов из `lists\*.txt` с прогресс-баром и итогами.
- **Сетевой монитор** в реальном времени: живые TCP/UDP соединения, гео, фильтр,
  сортировка, пауза, без мерцания.
- Двуязычный интерфейс (RU/EN), переключение на лету.
- Логи для техподдержки (внешний IP, гео, все статусы) в `Logs\`.
- Переключаемая легенда статусов прямо в меню (кнопка `L`).

## 5. Архитектура и технологии

- **Язык:** PowerShell 5.1 (совместим с .NET 4.x). Один файл, `xcopy`-деплой.
- **Сеть:** низкоуровневый `TcpClient`/`SslStream` — чтобы видеть RST и таймауты
  по отдельности, а не общий «fail» от высокоуровневых командлетов.
- **Сертификаты:** `X509Certificate2` / `X509Chain`, кастомный C#-делегат
  `TrustAllCerts` (через `Add-Type`) для перехвата проверки в SslStream.
- **UI:** ручная отрисовка в консоли. Рамки строятся функцией `Format-Box`
  (программное выравнивание), двухколоночный рендер — `Write-TwoColumns`.
- **Без внешних модулей** — только встроенные командлеты и .NET.

Подробная карта функций — в [TECHNICAL.md](TECHNICAL.md).

## 7. Быстрый старт

1. Распаковать в любую папку.
2. Двойной клик по **`NetworkChecker.bat`** (или ПКМ по `.ps1` →
   «Запустить с помощью PowerShell»). Скрипт сам запросит права администратора.
3. В меню: `6` — проверить домен, `2` — проверить список, `1` — монитор.
4. После проверки `S` — сохранить отчёт в `Logs\`.

Полное руководство — в [MANUAL.md](MANUAL.md).

## 8. Ограничения и нюансы

- **Только Windows** (PowerShell 5.1, `netstat`, Windows API).
- **ICMP** часто режется — `PING:timeout` ≠ недоступность; ориентируйтесь на
  TLS/HTTP.
- **MITM** может быть вызван вашим антивирусом (Kaspersky/ESET перехватывают TLS),
  а не провайдером.
- Колонка **KB/s** в мониторе — заглушка (реальный per-process трафик через
  perf-счётчики слишком медленный для 3-сек тика); см. идею в
  [BUGS_AND_FIXES.md](BUGS_AND_FIXES.md).
- Интерфейс рассчитан на терминал **шириной ≥ 80 столбцов**.
